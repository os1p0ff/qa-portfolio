# 2. Анализ функциональности Charles Proxy

## Фильтрация запросов (Filter)

### Назначение
Позволяет отображать только нужные запросы среди всего сетевого трафика по URL, типу, домену или содержимому.

### Практическое применение
Тестировщик использует фильтрацию для:
- поиска запросов конкретного API;
- анализа загрузки отдельных ресурсов;
- проверки корректности отправки запросов;
- ускорения поиска ошибок.

### Пример выполнения
Во время работы использовался фильтр для поиска запросов, связанных с изображениями и карточкой товара.

---

## Работа с логами (Logs)

### Назначение
Позволяет сохранять историю запросов и ответов между клиентом и сервером.

### Практическое применение
Тестировщик использует логи для:
- воспроизведения ошибок;
- анализа последовательности действий пользователя;
- проверки статусов ответа сервера;
- поиска причины некорректного поведения сайта.

### Пример выполнения
Во время выполнения задания анализировались запросы загрузки страницы товара и ответы сервера.

---

## Работа с зашифрованным трафиком (SSL Proxying)

### Назначение
Позволяет расшифровывать HTTPS-трафик и просматривать содержимое запросов и ответов.

### Практическое применение
Тестировщик использует SSL Proxying для:
- анализа API;
- просмотра JSON-ответов;
- проверки передаваемых данных;
- поиска ошибок интеграции.

### Пример выполнения
Во время работы был включён SSL Proxying для просмотра содержимого HTTPS-запросов интернет-магазина.

---

## Map Remote / Map Local

### Назначение
Позволяет заменить ответ сервера локальным файлом или перенаправить запрос.

### Практическое применение
Тестировщик использует функцию для:
- проверки отображения интерфейса без изменений на сервере;
- тестирования отсутствующих ресурсов;
- подмены изображений и файлов.

### Пример выполнения
В ходе задания была выполнена замена изображения товара на локальное изображение через Map Local.

---

## Breakpoint

### Назначение
Позволяет остановить выполнение запроса или ответа и изменить данные вручную.

### Практическое применение
Тестировщик использует Breakpoint для:
- проверки реакции интерфейса на изменённые данные;
- проверки обработки ошибок;
- моделирования нестандартных ответов сервера.

### Пример выполнения
Во время задания было изменено название товара непосредственно в ответе сервера.

---

## Rewrite

### Назначение
Позволяет автоматически изменять запросы и ответы без ручного вмешательства.

### Практическое применение
Тестировщик использует Rewrite для:
- изменения цен;
- изменения статусов ответа;
- тестирования различных сценариев работы клиента.

### Пример выполнения
Во время выполнения задания была автоматически изменена цена товара через Rewrite.

---

## Throttling

### Назначение
Позволяет искусственно ограничить скорость соединения.

### Практическое применение
Тестировщик использует Throttling для:
- проверки поведения сайта при плохом интернете;
- оценки скорости загрузки;
- тестирования отображения интерфейса.

### Пример выполнения
Во время задания был выбран режим медленного соединения (3G), после чего была проверена загрузка страницы.
