# Отчет по анализу рисков безопасности веб-приложения «Безопасный чат для врачей и пациентов»

## Введение

Анализируемое приложение представляет собой веб-чат для обмена сообщениями между врачами и пациентами. Через чат передаются персональные данные пользователей, медицинские заключения, результаты анализов, назначения и другие сведения, относящиеся к медицинской тайне.

Поскольку приложение работает с конфиденциальной информацией, необходимо провести анализ потенциальных угроз безопасности и определить риски нарушения принципов конфиденциальности, целостности и доступности информации.

# Топ-3 критических риска

## 1. Несанкционированный доступ к чужому чату

**Причина выбора:**
- Прямая утечка медицинской тайны.
- Нарушение законодательства о защите персональных данных.
- Высокий ущерб для пациентов и репутации компании.

## 2. Захват учетной записи врача или пациента

**Причина выбора:**
- Позволяет злоумышленнику читать всю переписку.
- Может привести к изменению медицинских рекомендаций.
- Часто встречается на практике.

## 3. Изменение сообщений через уязвимости API

**Причина выбора:**
- Может привести к ошибочному лечению пациента.
- Нарушает доверие к системе.
- Влияет на целостность медицинской информации.

# Рекомендации для разработчиков

## Риск 1. Несанкционированный доступ к чужому чату

**Меры снижения риска:**
1. Реализовать строгую проверку прав доступа на сервере.
2. Проводить регулярное тестирование авторизации и контроля доступа.

## Риск 2. Захват учетной записи врача или пациента

**Меры снижения риска:**
1. Внедрить многофакторную аутентификацию (MFA).
2. Настроить ограничения на количество попыток входа и мониторинг подозрительной активности.

## Риск 3. Изменение сообщений через уязвимости API

**Меры снижения риска:**
1. Выполнять валидацию всех входящих запросов на сервере.
2. Использовать журналирование действий пользователей и контроль целостности данных.
