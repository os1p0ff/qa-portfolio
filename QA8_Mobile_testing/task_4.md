# Анализ логов и логирования

## Введение

Во время тестирования мобильного приложения онлайн-банка была обнаружена проблема: при попытке открыть историю операций за последний месяц приложение аварийно завершается (crash).

Для поиска причины необходимо проанализировать логи приложения и системы.

---

## 1. Какие уровни логов наиболее важны для анализа проблемы

Для расследования аварийного завершения приложения наиболее полезными являются следующие уровни логирования:

| Уровень | Назначение | Обоснование |
|----------|------------|-------------|
| ERROR (E) | Ошибки, приводящие к сбоям приложения | Позволяет быстро найти причину аварийного завершения |
| WARN (W) | Предупреждения о потенциальных проблемах | Может показать проблему до возникновения сбоя |
| INFO (I) | Информация о работе приложения | Помогает восстановить последовательность действий пользователя |
| DEBUG (D) | Подробная отладочная информация | Показывает детали выполнения операций и запросов |

### Наиболее важные уровни

Для данной проблемы основной интерес представляют:

- ERROR (E)
- WARN (W)
- DEBUG (D)

Именно они чаще всего содержат информацию об исключениях, ошибках обработки данных и причинах падения приложения.

---

## 2. Механизм получения логов

Для Android-приложений используется инструмент Logcat в Android Studio.

### Базовые шаги

1. Запустить приложение на устройстве или эмуляторе.
2. Открыть Android Studio.
3. Перейти в окно:

   ```text
   View → Tool Windows → Logcat
   ```

4. Выбрать нужное устройство.
5. Воспроизвести проблему.
6. Найти сообщения об ошибках.
7. Сохранить или приложить необходимые логи к баг-репорту.

---

## 3. Какие фильтры использовать в Logcat

Для ускорения поиска причины проблемы можно применять фильтрацию логов.

### Фильтр по уровню ошибок

```text
level:ERROR
```

Показывает только сообщения об ошибках.

### Фильтр по пакету приложения

```text
package:com.bank.mobile
```

Показывает только логи банковского приложения.

### Фильтр по процессу

```text
process:com.bank.mobile
```

Отображает сообщения только выбранного процесса.

### Поиск по ключевому слову

```text
Exception
```

Позволяет быстро найти исключения.

### Поиск NullPointerException

```text
NullPointerException
```

Частая причина аварийного завершения приложения.

### Поиск OutOfMemoryError

```text
OutOfMemoryError
```

Используется для поиска проблем с памятью.

### Поиск по экрану истории операций

```text
TransactionHistory
```

или

```text
HistoryFragment
```

Позволяет отфильтровать сообщения, относящиеся к экрану истории операций.

### Комбинированный фильтр

```text
package:com.bank.mobile level:ERROR
```

Показывает только ошибки банковского приложения.

## 4. Драфт баг-репорта

### Заголовок

Crash при открытии истории операций за последний месяц

### ID

BUG-001

### Severity

Critical

### Priority

High

### Окружение

- Android 17
- Pixel 10 Emulator
- Версия приложения: 1.0.0
- Wi-Fi подключение

### Предусловия

Пользователь авторизован в приложении.

### Шаги воспроизведения

1. Запустить приложение.
2. Выполнить вход в аккаунт.
3. Перейти в раздел «История операций».
4. Выбрать период «Последний месяц».

### Фактический результат

Приложение аварийно завершается.

### Ожидаемый результат

Отображается список операций за последний месяц.
