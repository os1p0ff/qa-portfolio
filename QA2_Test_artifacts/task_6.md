# Задание 6. Тестовый сценарий
## Процесс: «Как поступить в Школу 21»

## 1. Кратко о различиях

### Сценарий использования (Use Case)

Описывает, как пользователь взаимодействует с системой, чтобы достичь цели.
Используется для описания требований к системе.

### Тестовый сценарий (Test Scenario)

Описывает, что именно нужно проверить в системе.
Используется тестировщиками для проверки работы функционала.

## 2. Сценарий использования: Поступление в Школу 21

### Кто использует систему?
Абитуриент (пользователь сайта).

### Цель пользователя
Подать заявку на обучение в Школу 21.

## 3. Предусловия

- У пользователя есть доступ к интернету.
- Сайт школы работает корректно.
- У пользователя есть email и номер телефона.

## 4. Основной сценарий (основной поток)

1. Пользователь открывает сайт Школы 21.
2. Нажимает кнопку «Подать заявку».
3. Система открывает форму регистрации.
4. Пользователь заполняет поля:
   - ФИО
   - Email
   - Номер телефона
5. Пользователь ставит галочку согласия на обработку персональных данных.
6. Нажимает кнопку «Отправить».
7. Система проверяет корректность данных.
8. Система отправляет подтверждение на email или телефон.
9. Пользователь подтверждает регистрацию.
10. Система выводит сообщение об успешной подаче заявки.

## 5. Альтернативные сценарии

### 5.1 Некорректный email
- Пользователь вводит email в неправильном формате.
- Система показывает сообщение об ошибке.
- Пользователь исправляет email.
- Регистрация продолжается.

### 5.2 Не заполнено обязательное поле
- Пользователь не заполняет одно из обязательных полей.
- Система подсвечивает поле и выводит ошибку.
- Пользователь заполняет поле.
- Регистрация продолжается.

### 5.3 Пользователь уже зарегистрирован
- Пользователь вводит уже существующий email.
- Система сообщает, что пользователь уже зарегистрирован.
- Предлагается восстановление доступа.

## 6. Ожидаемый результат

- Форма не отправляется при ошибках.
- Обязательные поля проверяются.
- Email и телефон валидируются.
- После корректного заполнения заявка успешно отправляется.
- Пользователь получает подтверждение регистрации.
