# Задание 1. Классы эквивалентности.

**Контекст:** Тестирование поля ввода логина на ресурсе по выдаче кредитов. Согласно ТЗ, ограничений на минимальное/максимальное количество символов нет.

## Определенные классы эквивалентности и тестовые значения

1.  **Класс: Допустимые буквенно-цифровые символы**
    *   *Описание:* Стандартные символы, которые гарантированно должны приниматься системой.
    *   *Тестовые значения:*
        *   `IvanPetrov`
        *   `user12345`

2.  **Класс: Специальные символы**
    *   *Описание:* Символы, которые могут использоваться в логинах, но требуют корректной обработки (экранирования) на бэкенде.
    *   *Тестовые значения:*
        *   `john.doe`
        *   `user+mail`
        *   `admin'--`

3.  **Класс: Пробельные символы**
    *   *Описание:* Наличие пробелов в разных частях строки. Важно для проверки корректности обрезки (trim) или сохранения данных.
    *   *Тестовые значения:*
        *   `"  leading_spaces"` (с пробелами в начале)
        *   `"trailing_spaces  "` (с пробелами в конце)
        *   `internal space` (с пробелом внутри)

4.  **Класс: Пустое значение**
    *   *Описание:* Проверка обработки отсутствия ввода.
    *   *Тестовые значения:*
        *   `""` (пустая строка)

5.  **Класс: Не-ASCII символы**
    *   *Описание:* Проверка поддержки национальных алфавитов.
    *   *Тестовые значения:*
        *   `ИванПетров`
        *   `用户123`

6.  **Класс: Управляющие символы**
    *   *Описание:* Специальные непечатаемые символы, которые могут нарушить логику работы системы.
    *   *Тестовые значения:*
        *   Строка с символом табуляции: `Name\tSurname` (при вставке или отправке запроса)
