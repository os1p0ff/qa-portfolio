# Задание 3. Базы данных

## Выбранный сервис

Telegram — популярный мессенджер для обмена сообщениями, файлами и медиа-контентом.

## Гипотеза о типе БД

По моему мнению, Telegram использует гибридный подход к хранению данных, сочетая реляционные и нереляционные базы данных.

Реляционные БД подходят для хранения информации о пользователях, настройках аккаунта и связях между объектами.

Нереляционные БД лучше подходят для хранения большого количества сообщений, медиафайлов и быстрого масштабирования системы.

---

## Ключевые функции сервиса и предполагаемые типы БД

### 1. Авторизация и профиль пользователя

Используемые данные:
- номер телефона;
- имя пользователя;
- данные авторизации;
- коды подтверждения;
- настройки аккаунта.

Предполагаемый тип БД:
Реляционная БД.

Почему:
Данные пользователей имеют четкую структуру и связи между таблицами. Для таких данных важны надежность, целостность и поддержка транзакций.

---

### 2. Обмен сообщениями

Используемые данные:
- текст сообщений;
- история переписки;
- время отправки;
- статусы сообщений.

Предполагаемый тип БД:
Нереляционная БД.

Почему:
Telegram хранит огромное количество сообщений. Нереляционные БД лучше подходят для быстрого масштабирования и обработки больших объемов данных.

---

### 3. Хранение медиафайлов

Используемые данные:
- фотографии;
- видео;
- документы;
- голосовые сообщения.

Предполагаемый тип БД:
Нереляционная БД.

Почему:
Медиафайлы имеют большой размер и могут храниться в распределенных системах хранения данных. Нереляционные решения лучше подходят для таких задач.

---

### 4. Группы и каналы

Используемые данные:
- список участников;
- роли пользователей;
- настройки группы;
- сообщения канала.

Предполагаемый тип БД:
Гибридный подход.

Почему:
Информация об участниках и правах доступа требует строгой структуры, а сообщения и контент каналов удобнее хранить в нереляционных БД.

---

## Идеи тест-кейсов для проверки работы БД

### Тест-кейс 1. Проверка сохранения сообщений

Шаги:
1. Отправить сообщение другому пользователю.
2. Перезагрузить приложение.
3. Проверить наличие сообщения в истории переписки.

Ожидаемый результат:
Сообщение успешно сохраняется и отображается после повторного открытия приложения.

---

### Тест-кейс 2. Проверка целостности данных профиля

Шаги:
1. Изменить имя пользователя и фотографию профиля.
2. Выйти из аккаунта.
3. Повторно войти в аккаунт.

Ожидаемый результат:
Измененные данные профиля сохраняются корректно.

---

### Тест-кейс 3. Проверка загрузки медиафайлов

Шаги:
1. Отправить видеофайл в чат.
2. Открыть чат с другого устройства.
3. Проверить доступность файла.

Ожидаемый результат:
Медиафайл успешно хранится и доступен на всех устройствах.
