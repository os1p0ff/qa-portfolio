# Задание 1. Классификация тестирования

## Таблица категоризации типов тестирования

| По стадии проведения процесса тестирования | По знанию устройства системы | По направленности тестового сценария | По выполнению кода |
|---|---|---|---|
| **Alpha-тестирование:** тестирование макета интерфейса силами дизайнеров и продукт-менеджеров на удобство и логику переходов между экранами. | **Black-box:** проверка перевода средств между счетами без знания внутренней логики кода. Проверяется правильность ввода суммы и отображение результата перевода. | **Функциональное тестирование:** проверка, что перевод между счетами работает и деньги корректно списываются и зачисляются. | **Ручное тестирование:** тестировщик вручную проверяет сценарий перевода денег между счетами. |
| **Beta-тестирование:** тестирование первой полной сборки приложения сотрудниками компании. | **Grey-box:** тестирование перевода средств с частичным знанием системы (например, понимание структуры базы счетов). | **Нефункциональное тестирование:** проверка скорости загрузки экранов приложения. | **Автоматизированное тестирование:** написание автотеста для проверки перевода между счетами. |
| **Smoke-тестирование:** проверка основных функций после сборки приложения (запуск приложения, вход, переход на экран перевода). | **White-box:** разработчик проверяет корректность логики функции перевода в коде. | **Тестирование безопасности:** проверка, что нельзя перевести отрицательную сумму или обойти ограничения. | **Полуавтоматическое тестирование:** часть тестов выполняется автоматически, часть вручную. |
| **Регрессионное тестирование:** повторная проверка работы перевода средств после исправления ошибок. |  | **Тестирование удобства (Usability):** проверка удобства интерфейса перевода денег. |  |
