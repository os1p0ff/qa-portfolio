## 1. Этапы установления HTTPS-соединения

### 1. Клиент отправляет запрос на подключение

Браузер отправляет серверу запрос на установление защищенного соединения.

**Цель этапа:**
Начать процесс защищенного обмена данными.

**Какой риск устраняет:**
Без этого этапа защищенное соединение невозможно.

---

### 2. Сервер отправляет SSL/TLS-сертификат

Сервер отправляет браузеру сертификат безопасности.

**Цель этапа:**
Подтвердить подлинность сайта.

**Какой риск устраняет:**
Защита от поддельных сайтов и MITM-атак.

---

### 3. Браузер проверяет сертификат

Браузер проверяет:
- срок действия сертификата;
- доверенный центр сертификации;
- соответствие домену.

**Цель этапа:**
Убедиться, что соединение безопасно.

**Какой риск устраняет:**
Предотвращение подключения к вредоносному серверу.

---

### 4. Согласование ключей шифрования

Браузер и сервер создают общий секретный ключ.

**Цель этапа:**
Настроить шифрование данных.

**Какой риск устраняет:**
Перехват передаваемой информации.

---

### 5. Установка защищенного TLS-соединения

После успешного обмена ключами создается защищенный канал связи.

**Цель этапа:**
Передавать данные в зашифрованном виде.

**Какой риск устраняет:**
Кража логинов, паролей и других данных.

---

### 6. Передача данных по HTTPS

Браузер и сервер начинают обмен зашифрованными HTTP-запросами.

**Цель этапа:**
Безопасная работа сайта.

**Какой риск устраняет:**
Подмена и чтение трафика третьими лицами.

## 3. HTTP Status Codes Cheat Sheet

### 1xx — Информационные ответы

| Код | Что видит пользователь | Что происходит | Как тестировать | Куда эскалировать |
|------|-----------------------|----------------|-----------------|-------------------|
| 100 Continue | Обычно ничего | Сервер готов принять запрос | Проверить upload больших файлов | Backend |

### 2xx — Успешные ответы

| Код | Что видит пользователь | Что происходит | Как тестировать | Куда эскалировать |
|------|-----------------------|----------------|-----------------|-------------------|
| 200 OK | Страница работает | Запрос успешно обработан | Проверить корректность данных | Backend |
| 201 Created | Объект создан | Сервер создал ресурс | Проверить POST-запросы | Backend |

### 3xx — Перенаправления

| Код | Что видит пользователь | Что происходит | Как тестировать | Куда эскалировать |
|------|-----------------------|----------------|-----------------|-------------------|
| 301 Moved Permanently | Переход на другой URL | Постоянный редирект | Проверить redirect | Frontend/Backend |
| 302 Found | Временное перенаправление | Временный redirect | Проверить авторизацию | Frontend |

### 4xx — Ошибки клиента

| Код | Что видит пользователь | Что происходит | Как тестировать | Куда эскалировать |
|------|-----------------------|----------------|-----------------|-------------------|
| 400 Bad Request | Ошибка запроса | Неверный формат запроса | Отправить неправильные данные | Backend |
| 401 Unauthorized | Требуется вход | Нет авторизации | Проверить токены | Backend |
| 403 Forbidden | Доступ запрещен | Нет прав | Проверить роли | Backend |
| 404 Not Found | Страница не найдена | Ресурс отсутствует | Открыть несуществующий URL | Frontend/Backend |

### 5xx — Ошибки сервера

| Код | Что видит пользователь | Что происходит | Как тестировать | Куда эскалировать |
|------|-----------------------|----------------|-----------------|-------------------|
| 500 Internal Server Error | Ошибка сервера | Сбой backend | Вызвать некорректный запрос | Backend |
| 502 Bad Gateway | Сервер недоступен | Проблемы между серверами | Проверить API gateway | DevOps |
| 503 Service Unavailable | Сервис временно недоступен | Перегрузка/maintenance | Проверить retry | DevOps |

## 4. Практические HTTP-запросы

### Сценарий 1: Страница не найдена → статус 404 Not Found
Что делаю?	Отправляю GET-запрос на https://httpbin.org/status/404
Какая ситуация?	Пользователь ввел в адресной строке несуществующий URL (например, https://site.ru/ne-sushhestvuyu). Такой страницы на сервере нет
Какой статус?	404 Not Found — запрошенный ресурс отсутствует на сервере

### Сценарий 2: Ошибка сервера → статус 500 Internal Server Error
Что делаю?	Отправляю GET-запрос на https://httpbin.org/status/500
Какая ситуация?	Пользователь пытается зайти на страницу, но на сервере произошел сбой (например, упала база данных или ошибка в коде). Сервер не может обработать запрос
Какой статус?	500 Internal Server Error — сервер столкнулся с непредвиденной ошибкой и не может выполнить запрос
